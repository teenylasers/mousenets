1.2. Architecture

A multilayer perceptron (MLP) is a fully connected network that sequentially consists of the input layer, the hidden layer(s), and the output layer. Each hidden or output layer is given by

\[\begin{align} H^{(i)} &= W^{(i)}X^{(i-1)} \\ X^{(i)} &= F^{(i)}(H^{(i)}) \end{align}\]

for the \(i\)-th layer, where \(X^{(i-1)}\) is the output from the previous layer and the input to the current one, \(W^{(i)}\) is the weights matrix, and \(F^{(i)}\) is the activation function, typically sigmoid, ReLU, or softmax. A loss function is used to compare the MLP output \(X^(n)\) with the ground truth \(Y\) and quantify its goodness,

\[\begin{align} \mathcal{L}(X^{(n)}, Y) : \mathbb{R}^K \rightarrow \mathbb{R} \end{align}\]

We use superscripts in parentheses to denote layer number, and subscripts for matrix manipulation using Einstein summation notation.

When we look at a layer on its own, we simplify the notation by ridding of the superscript, and rename the layer output \(Y\),

\[\begin{align} H &= WX \\ Y &= F(H) \end{align}\]

where \(Y\) is an \(N\)-dimensional vector, \(X\) is an \(N_x\)-dimensional vector, and \(W\) is an \((N \times N_x)\) matrix.

1.3. Example implementations

1.3.1. MLP(32,10) for MNIST

For a first implementation to classify hand-written digits using the MNIST dataset, we use a 3-layer MLP. Each input image is \(28\times28\) pixels, converted to a 784-element vector for the input layer. The single hidden layer is 32-wide, using the sigmoid activation function. The output layer is 10-wide, using the softmax activation function. Therefore,

\[\begin{align} X_0 &\in \mathbb{R}^{784 \times 1} \nonumber \\ W_1 &\in \mathbb{R}^{785 \times 32} \nonumber \\ X_1 &\in \mathbb{R}^{32 \times 1} \nonumber \\ W_2 &\in \mathbb{R}^{33 \times 10} \nonumber \\ X_2 &\in \mathbb{R}^{10 \times 1} \end{align}\]

Categorical cross-entropy is used to calculate loss.

1.4. Activation function

1.4.1. Sigmoid function

\[\begin{align} \sigma(H) &= \frac{1}{1+e^H} \label{eq:sigmoid} \\ \frac{\partial\sigma}{\partial H} &= \sigma (1-\sigma) \end{align}\]

1.4.2. Softmax function

\[\begin{align} s_j(H) &= \frac{e^{H_j}}{\sum_i^{K} e^{H_i}} \label{eq:softmax} \\ \frac{\partial s_j}{\partial H_k} &= \begin{cases} \frac{e^{H_k}(\sum_l e^{H_l} - e^{H_k})}{(\sum_l e^{H_l})^2}, & j=k \\ \frac{-e^{H_j + H_k}}{(\sum_l e^{H_l})^2}, & j \neq k \end{cases} \nonumber \\ &= \begin{cases} s(H_k)(1-s(H_k)), & j=k \\ \frac{-e^{H_j + H_k}}{(\sum_l e^{H_l})^2}, & j \neq k \end{cases} \nonumber \\ &= \text{Diag}(s) - ss^T \label{eq:softmax_grad} \end{align}\]

1.5. Loss function

1.5.1. Categorial cross-entropy loss

Softmax function is applied to interpret neural network output as the probability that the input classifies to an output class \(j\), \(P(Y_j | X)\). The sum of the probabilities across the classes is 1, thus the input must be of one of the output categories. Apply maximum likelihood estimation, we can minimize the negative log-likelihood, which is equivalent to maximizing the probability. Thus the cross-entropy loss function becomes

\[\begin{align} \mathcal{L}_{CE} &= -\sum_i Y_i \log(X^{(n)}_i) \label{eq:cross_entropy_loss} \\ \frac{\partial\mathcal{L}_{CE}}{\partial X} &= -\frac{Y}{X} \end{align}\]

When \(Y\) is a one-hot encoded vector, categorial cross-entropy loss is

\[\begin{align} \mathcal{L}_{CCE} &= -\log(x_p) \\ \frac{\partial \mathcal{L}_{CCE}}{\partial X_j} &= \begin{cases} 0, & Y_j = 0 \\ -\frac{Y_j}{X_j}, & Y_j = 1 \end{cases} \end{align}\]

When categorical cross-entropy loss is combined with softmax function as the activation function of the last layer, we can combine the loss gradient calculation into a single simplified calculation. We substitute \eqref{eq:softmax} into \eqref{eq:cross_entropy_loss}

\[\begin{align} \mathcal{L}_{SCE}(H) &= -\sum_i Y_i \log \frac{e^{H_i}}{\sum_k e^{H_k}} \nonumber \\ &= -\sum_i Y_i \log e^{H_i} + \sum_i Y_i \log \sum_k e^{H_k} \nonumber \\ &= -\sum_i Y_i H_i + \log \sum_k e^{H_k} \label{eq:loss_sce} \\ \frac{\partial \mathcal{L}_{SCE}}{\partial H_i} &= -Y_i + \frac{e^{H_i}}{\sum_k e^{H_k}} \nonumber \\ &= -Y_i + \text{softmax}(H_i) \label{eq:loss_grad_sce} \end{align}\]

\eqref{eq:loss_sce} and \eqref{eq:loss_grad_sce} simplify backpropagation calculations in gradient descent, letting us bypass the somewhat complex \eqref{eq:softmax_grad}.

1.6. Backpropagation

We want to minimize the loss function \(\mathcal{L}\) using gradient descent. From the neural net's output \(X_n\) and the ground truth \(Y\), we can calculate how \(\mathcal{L}\) changes due to each component of \(X_n\), i.e. the gradient \(\partial\mathcal{L} / \partial X_n\). We apply the chain rule and work the loss gradient backwards through the net. Writing out the last two layers, we have

\[\begin{align*}H^{(n-1)} = W^{(n-1)}X^{(n-2)}\end{align*}\]
\[\begin{align*}X^{(n-1)} = F^{(n-1)}(H^{(n-1)}) \nonumber\end{align*}\] \[\begin{align*}\frac{\partial X^{(n-1)}}{\partial H^{(n-1)}}\end{align*}\] \[\begin{align*}\frac{\partial\mathcal{L}}{\partial H^{(n-1)}} = \frac{\partial\mathcal{L}}{\partial X^{(n-1)}} \frac{\partial X^{(n-1)}}{\partial H^{(n-1)}}\end{align*}\]
\[\begin{align*}H^{(n)} = W^{(n)}X^{(n-1)}\end{align*}\] \[\begin{align*}\frac{\partial H^{(n)}}{\partial W^{(n)}}, \frac{\partial H^{(n)}}{\partial X^{(n-1)}}\end{align*}\] \[\begin{align*} \frac{\partial\mathcal{L}}{\partial W^{(n)}} = \frac{\partial\mathcal{L}}{\partial H^{(n)}} \frac{\partial H^{(n)}}{\partial W^{(n)}}, \frac{\partial\mathcal{L}}{\partial X^{(n-1)}} = \frac{\partial\mathcal{L}}{\partial H^{(n)}} \frac{\partial H^{(n)}}{\partial X^{(n-1)}}\end{align*}\]
\[\begin{align*}X^{(n)} = F^{(n)}(H^{(n)})\end{align*}\] \[\begin{align*}\frac{\partial X^{(n)}}{\partial H^{(n)}} = F'^{(n)}(H^{(n)})\end{align*}\] \[\begin{align*} \frac{\partial\mathcal{L}}{\partial H^{(n)}} = \frac{\partial \mathcal{L}}{\partial X^{(n)}} \frac{\partial X^{(n)}}{\partial H^{(n)}}\end{align*}\]
\[\begin{align*}\mathcal{L}(X^{(n)}, Y)\end{align*}\] \[\begin{align*}\frac{\partial\mathcal{L}}{\partial X^{(n)}}\end{align*}\]

Thus, for each \(i\)-th layer, backpropagation is given by

\[\begin{align} \frac{\partial \mathcal{L}}{\partial W^{(i)}} &= \frac{\partial X^{(i)}}{\partial H^{(i)}} \frac{\partial \mathcal{L}}{\partial X^{(i)}} \left(\frac{\partial H^{(i)}}{\partial W^{(i)}}\right)^T \nonumber \\ &= \frac{\partial X^{(i)}}{\partial H^{(i)}} \frac{\partial \mathcal{L}}{\partial X^{(i)}} (X^{(i-1)})^T \label{eq:backprop_weights} \\ (N \times N_x) &= (N \times N) (N \times 1) (1 \times N_x) \nonumber \end{align}\] \[\begin{align} \frac{\partial \mathcal{L}}{\partial X^{(i-1)}} &= \left(\frac{\partial H^{(i)}}{\partial X^{(i-1)}}\right)^T \frac{\partial X^{(i)}}{\partial H^{(i)}} \frac{\partial \mathcal{L}}{\partial X^{(i)}} \nonumber \\ &= (W^{(i)})^T \frac{\partial X^{(i)}}{\partial H^{(i)}} \frac{\partial \mathcal{L}}{\partial X^{(i)}} \label{eq:backprop_x} \\ (N_x \times 1) &= (N_x \times N) (N \times N) (N \times 1) \nonumber \end{align}\]

where \eqref{eq:backprop_weights} updates the weights in this layer and \eqref{eq:backprop_x} propagates the loss error to the previous layer. \eqref{eq:backprop_weights} and \eqref{eq:backprop_x} are written in the form to perform backpropagation using matrix operations. For checking matrix dimensions, \(N\) is the width, i.e. the number of nodes in this layer, it is also the width of the layer output; \(N_x\) is the width of the input to this layer.

1.6.1. Check gradient

Backpropagation \eqref{eq:backprop_weights} and \eqref{eq:backprop_x} gives us analytically calculated gradient at each layer of the neural network. We can check the correctness of our implementation against numerically calculate gradients, that is, for the \(i\)-th layer, we wish to verify the loss gradient with respect to the layer input \(\frac{\partial \mathcal{L}}{\partial X^{(i-1)}}\) and the weights \(\frac{\partial \mathcal{L}}{\partial W^{(i)}}\)

We can perform gradient check at each layer individually, calculating \(\frac{\partial \mathcal{L}}{\partial X}\) and \(\frac{\partial \mathcal{L}}{\partial W}\) from \(\frac{\partial \mathcal{L}}{\partial Y}\),

\[\begin{align} \frac{\partial \mathcal{L}}{\partial X_k} &= \sum^N_j \frac{\partial \mathcal{L}}{\partial Y_j} \frac{\partial Y_j}{\partial X_k} \nonumber \nonumber \\ &= \sum^N_j \frac{\partial \mathcal{L}}{\partial Y_j} \frac{Y_j(X_k + \epsilon) - Y_j(X_k - \epsilon)}{2\epsilon} \end{align}\]

Alternatively, we can check gradient by tweaking the \(k\)-th element on the \(i\)-th layer in the neural net \(X^{(i)}_k\), then forward propagate its effect through the network all the way to the loss function \(\mathcal{L}\) to calculate the loss gradient,

\[\begin{align} \frac{\partial \mathcal{L}}{\partial X^{(i)}_k} &= \frac{\mathcal{L}(X^{(i)}_k + \epsilon) - \mathcal{L}(X^{(i)}_k - \epsilon)}{2\epsilon} \end{align}\]

1.7. Optimization

1.7.1. Gradient descent: stochastic and batch

The loss function is defined as an evaluation over the whole data set. From that point of view, we would only backpropagate loss gradient after the entire data set. However, this proves to be inefficient; we need not look at the entire data set to get a good estimate of the loss gradient and update the weights. The opposite strategy is to use stochastic gradient descent, where the weights are updates after each sample. This strategy can be noisy. A middle ground is to use minibatches, where we average over enough samples to smooth out the per-sample noise, yet not so many that the advantage of averaging has diminishing returns.

To implement batch gradient descent, we backpropagate the loss gradient for each sample, calculate and accumulate \(\frac{\partial \mathcal{L}}{\partial W}\). At the end of each batch, we average \(\frac{\partial \mathcal{L}}{\partial W}\) over the batch size and update the weights.

Alternatively, some implementations cache \(X^{(i)}\) for all samples, and perform backpropagation once over the \((N_x \times \text{batch_size})\) \(X^{(i)}\) matrices. This requires more memory to save intermediate results and matrix by matrix operations for analytical gradient calculations.

2. ConvNet

2.1. Architecture

2.1.1. Convolutional layer

For an input with the dimension \(N_x^{(i-1)} \times N_y^{(i-1)} \times C^{(i-1)}\), it is convolved with a kernel of size \(K \times K \times C^{(i-1)}\). Both the input and the kernelt have the same depth, or number of channels, \(C^{(i-1)}\) The convolutional layer is defined by the following hyperparameters,

  • \(C\), the number of filters

  • \(K\), filter kernel size, where \(K \geq 1\)

  • \(S\), stride, displacement between kernel scans

  • \(P\), zero-padding around the edges of the input

This results in an output of dimensions \(N_x^{(i)} \times N_y^{(i)} \times C^{(i)}\), where

\[\begin{align} N_x^{(i)} &= (N_x^{(i-1)} - K + 2P)/S + 1 \nonumber \\ N_y^{(i)} &= (N_y^{(i-1)} - K + 2P)/S + 1 \\ C^{(i)} &= C \nonumber \end{align}\]

Each filter has \(K^2C^{(i-1)}+1\) parameters, \(+1\) for the bias \(b\). The number of parameters to train in this layer is \(C^{(i)}(K^2C^{(i-1)}+1)\).

Note that the same \(K \times K\) kernel is shared across the entire input image for that filter layer. During backpropagation, every neuron in the volume will compute the gradient for its kernel weights, they are then averaged to update the single kernel.

Using similar notations as the fully-connected layer, we call the kernel the weight matrix \(W\), the output of the convolutional layer \(H\). For the \(i\)-th layer,

\[\begin{align} H^{(i)}_{ij} = \sum_{p=1}^K \sum_{q=1}^K \sum_{r=1}^{C^{(i-1)}} W^{(i)}_{pqr} X^{(i-1)}_{Si+p-1, Sj+q-1, r} + b^{(i)} \end{align}\]

where \(b\) is the bias, and this convolution is repeated for the \(c = 1..C^{(i)}\) kernels in this layer. The complete convolution for in this layer is then

\[\begin{align} \label{eq:conv_layer} H^{(i)}_{cij} = \sum_{p=1}^K \sum_{q=1}^K \sum_{r=1}^{C^{(i-1)}} W^{(i)}_{cpqr} X^{(i-1)}_{Si+p-1, Sj+q-1, r} + b^{(i)}_c \end{align}\]

Locally-connected layer

Sometimes it does not make sense to use the same convolutional kernel across the entire image. When different kernel weights are used in different regions of the image, the layer is typically called a locally-connected layer.

Backpropagation

To backpropagate the loss gradient \(\partial \mathcal{L} / \partial H^{(i)}\) to the weights \(\partial \mathcal{L} / \partial W\) and the biases \(\partial \mathcal{L} / \partial b\), we apply the chain rule on \eqref{eq:conv_layer},

\[\begin{align} \frac{\partial \mathcal{L}}{\partial W^{(i)}_{cpqr}} &= \sum_i \sum_j \frac{\partial \mathcal{L}}{\partial H^{(i)}_{cij}} \frac{\partial H^{(i)}_{cij}}{\partial W^{(i)}_{cpqr}} \nonumber \\ &= \sum_i \sum_j \frac{\partial \mathcal{L}}{\partial H^{(i)}_{cij}} X^{(i-1)}_{Si+p-1, Sj+q-1, r} \end{align}\] \[\begin{align} \frac{\partial \mathcal{L}}{\partial b^{(i)}_c} = \sum_i \sum_j \frac{\partial \mathcal{L}}{\partial H^{(i)}_{cij}} \frac{\partial H^{(i)}_{cij}}{\partial b^{(i)}_c} = \sum_i \sum_j \frac{\partial \mathcal{L}}{\partial H^{(i)}_{cij}} \end{align}\]

To get \(\partial \mathcal{L} / \partial X^{(i-1)}\), we rewrite the indices of \eqref{eq:conv_layer}

\[\begin{align} H^{(i)}_{cij} = \sum_{l} \sum_{m} \sum_{r} W^{(i)}_{c, l+1-Si, m+1-Sj, r} X^{(i-1)}_{lmr} + b^{(i)}_c \end{align}\]

then

\[\begin{align} \frac{\partial \mathcal{L}}{\partial X^{(i-1)}_{lmr}} &= \sum_i \sum_j \sum_c \frac{\partial \mathcal{L}}{\partial H^{(i)}_{cij}} \frac{\partial H^{(i)}_{cij}}{\partial X^{(i-1)}_{lmr}} \nonumber \\ &= \sum_i^{N_x^{(i)}} \sum_j^{N_y^{(i)}} \sum_c^{C^{(i)}} \frac{\partial \mathcal{L}}{\partial H^{(i)}_{cij}} W^{(i)}_{c,l+1-Si,m+1-Sj,r} \end{align}\]

2.1.2. ReLU layer

\[\begin{align} f_{\text{ReLU}}(x) = \max{(x,0)} \end{align}\] \[\begin{align} \frac{\partial f_{\text{ReLU}}}{\partial x} = \begin{cases} 1, x > 0 \\ 0, x \leq 0 \end{cases} \end{align}\]

2.1.3. Pooling layer

The pooling layer servers two purposes:

  1. mitigate the network's sensitivity to minor displacements in the input picture

  2. downsamples representation

It operates over each activation map independently. The pooling operation can be a max or averaging operation with a kernel size \(K\), with a stride \(S\). The output matrix is given by

\[\begin{align} Y_{cij} = \text{Pool} (X_{c, Si:Si+k, Sj:Sj+k}) \end{align}\]

where the function \(\text{Pool}(\cdot)\) can be \(\max(\cdot)\) or \(\text{mean}(\cdot)\)

The backpropagate the error gradient \(\partial \mathcal{L}/\partial Y\) to \(\partial \mathcal{L}/\partial X\),

\[\begin{align} \frac{\partial \mathcal{L}}{\partial X_{cpq}} &= \sum_i \sum_j \frac{\partial \mathcal{L}}{\partial Y_{cij}} \frac{\partial Y_{cij}}{\partial X_{cpq}} \end{align}\] \[\begin{align} \frac{\partial Y^{\text{max-pool}}_{cij}}{\partial X_{cpq}} &= \begin{cases} 1, Y_{cij} = X_{cpq} \\ 0, \text{otherwise} \end{cases} \end{align}\] \[\begin{align} \frac{\partial Y^{\text{average-pool}}_{cij}}{\partial X_{cpq}} &= \frac{X_{cpq}}{K^2} \end{align}\]

where \(c\) is the channel index, \(i = 0..N_x^{(i)}\), \(j = 0..N_y^{(i)}\), \(p = 0..N_x^{(i-1)}\), \(q = 0..N_y^{(i-1)}\).

2.1.4. Network architecture

ConvNets are stacks of convolutional, activation, pooling, and fully-connected (FC) layers. A traditional architecture lookes like

\[\begin{align} ( (\text{Conv} + \text{ReLU}) * N + \text{Pooling} ) * M + ( \text{FC} + \text{ReLU} ) * L + \text{Softmax} \end{align}\]

Relationship to fully-connected networks

For any convolutional layer, there is an equivalent fully-connected layer that implements the same forward function. Conversely, any fully-connected layer can be cast into an equivalent convolutional format by simple introducing a kernel that is the same size as the input image. Despite the mathematical equivalence, there are advantages to using convolutional layers:

  1. The ability to express fully-connected layers as a convolutional layer is useful. When the input image increases in size, we can use the same ConvNet to slide across the larger images in a single forward pass, instead of running the MLP over cropped images multiple times to achieve the same result.

  2. When expressing a convolutional layer as a fully-connected layer, the FC weights matrix contains many identical blocks due to weight-sharing of the kernels that concerns only local connectivity, and the large matrix is mostly zero. Thus, the conv layer contain many fewer parameters to train.

  3. Fully-connected architecture ignores the topology of the input. ConvNet gleams features from local structures in the input 2D images, and ensures some degree of shift, scale, and distortion invariance by its local receptive fields, shared weights, and spatial/temporal subsampling in pooling.

Deleting pooling layers

There are suggestions to remove pooling layers and downsample with conv layers using large strides.

2.2. Example implementations

2.2.1. Rewrite MLP(32,10) for MNIST as a ConvNet

We rewrite MLP(32,10) as a ConvNet. The resulting digit classification should perform the same. The layers in this ConvNet consist of:

  1. Conv: \(K=28\), \(S=1\), \(C=32\), filter depth is 1, \(28^2 \times 1 \times 32 + 32 = 25120\) trainable parameters.

  2. Activation: both sigmoid and ReLU are implemented.

  3. Conv: \(K=1\), \(S=1\), \(C=10\), filter depth is 32, \(1^2 \times 32 \times 10 + 10 = 330\) trainable paraemters.

  4. Softmax and cross-entropy loss.

2.2.2. LeNet-5 for MNIST classification

We implement LeNet-5 and test it on the MNIST data set. The layers in the network architecture are:

  1. Conv: kernel size \(K=5\), stride \(S=1\), depth \(C=6\), resulting in 6 feature maps, \(5^2 \times 6 + 6 = 156\) trainable parameters.

  2. Subsampling: \(K=2\), \(S=2\), \(C=6\), average-pooling, times coefficient, add bias, pass through the sigmoid function. There are \((1+1) \times 6 = 12\) trainable parameters, for the coefficient and bias in 6 layers.

  3. Conv: \(K=5\), \(S=1\), \(C=16\). The first 6 filters take from 3 contiguous subsampled feature maps, the next 6 from 4, the next 3 from 4 discontiguous subsampled feature maps, and the last filter from all 6. There are \(5^2 \times (3 \times 6 + 4 \times 6 + 4 \times 3 + 6) + 16 = 1516\) trainable parameters.

  4. Subsampling: \(K=2\), \(S=2\), \(C=16\), \((1+1)\times 16 = 12\) parameters.

  5. Conv/FC: \(K=5\), \(S=1\), \(C=120\), \(5^2 \times 16 \times 120 + 120 = 48120\) trainable connections.

  6. FC: 84 nodes, \(120 \times 84 + 84 = 10164\) parameters.

  7. Output layer: Euclidean Radial Basis Function units, one for each of the 10 classes, with 84 input each:

    \[\begin{align} y_i = \sum_j (x_j - w_{ij})^2 \end{align}\]

3. PointNet

3.1. Notes to sort and add

3.2. Architecture

  • Max pooling layer: a symmetric function to aggregate information from all of the points.

  • Local and global information combination structure

  • Two joint alignment networks to align both input points and point features

3.2.1. Symmetry function for unordered input

There are three strategies to make a model invariant to input permutation,

  1. Sort the input into a canonical order. This means to sort the input based on features that are intrinsic to the data as a preprocessing step, so that any input order will be pre-sorted according to the same criteria.

  2. Treat the unordered input as a sequence to train an RNN as the symmetrical function, augment the training data by all kinds of permutations

  3. Use a simple symmetric function to aggregate the information from each point, that is, the symmetric function produces an output vector that is invariance to the input vector's internal elements ordering.

The symmetric function approach is used in PointNet, as follows

\[\begin{align} f({x_1, ..., x_N}) \approx g(h(x_1), ... h(x_N)) \end{align}\]

where

\[\begin{align*} f : 2^{\mathbb{R}^N} &\mapsto \mathbb{R} \\ h : \mathbb{R}^N &\mapsto \mathbb{R}^K \\ g : \mathbb{R}^K \times ... \times \mathbb{R}^K &\mapsto \mathbb{R} \end{align*}\]

where \(N\) is the number of input points. \(h\) is approximated by a MLP. \(g\) is the symmetric function, implemented as a single variable function plus a max pooling function. The result \(f = [f_1, ..., f_K]\) after max pooling is a global feature of the input set. For classification, we can train an SVM or a MLP classifier on \(f\). However, point segmentation requires more network structures, addressed in the sections below.

3.2.2. Local and global information aggregation

Before the final MLP and max pooling to form the global feature, we have the transformed per-point features. Point segmentation requires knowledge of both local and global features. Therefore, the global feature is concatenated onto the local features.

3.2.3. Joint alignment network

More than set order invariance, semantic labeling of a point cloud needs to be invariant under certain geometric transformations, e.g. rigid body transformations. Again, a solution is to align the input set to a canonical space before feature extraction. PointNet does it by training an affine transformation matrix T-net. The same idea is applied to the alignment of feature space as well.

3.3. Example implementations

3.3.1. MNIST as a 2D point cloud, classification, no T-net

Following Appendix F in the PointNet paper, we transform the MNIST image data to pixel sets by intensity thresholding for values larger than 128. Each pixel contain the \((x,y)\) coordinates in the image. The set size is limited to 256. If there are more than 256 pixels after thresholding, randomly subsample to 256 points. If there are fewer than 256 pixels, then randomly repeat points from the set for padding.

3.3.2. MNIST, segmentation

For point segmentation, we will attempt to segment which pixels are the numbers and which are the background. With clean data, the segmentation is simply color based. To better test the PointNet implementation, we can add white or gray noise pixels and see how well segmentation performs.

4. Appendix

4.1. Matrix derivatives

4.1.1. Matrix-vector multiplication

\[\begin{align} y_{n \times 1} = A_{n \times m}x_{m \times 1} \\ f(y) : \mathbb{R}^{n \times 1} \mapsto \mathbb{R} \end{align}\] \[\begin{align} \frac{\partial f}{\partial x_j} = \frac{\partial f}{\partial y_i} \frac{\partial y_i}{\partial x_j} = A_{ji} \frac{\partial f}{\partial y_i} \end{align}\] \[\begin{align} \frac{\partial f}{\partial x} &= A^T \frac{\partial f}{\partial y} \\ (m \times 1) &= (m \times n)(n \times 1) \nonumber \end{align}\]

4.1.2. Elementwise vector function

\[\begin{align} y_{n \times 1} = \omega(x_{n \times 1}) \\ f(y) : \mathbb{R}^{n \times 1} \mapsto \mathbb{R} \end{align}\] \[\begin{align} \frac{\partial y_i}{\partial x_j} = \begin{cases} \omega'(x_j), & i=j \\ 0, & i \neq j \end{cases} = \text{diagonal matrix} \end{align}\] \[\begin{align} \frac{\partial f}{\partial x_j} = \frac{\partial y_i}{\partial x_j} \frac{\partial f}{\partial y_i} \end{align}\] \[\begin{align} \frac{\partial f}{\partial x} &= \omega'(x) \frac{\partial f}{\partial y} \\ (n \times 1) &= (n \times n)(n \times 1) \nonumber \end{align}\]

4.1.3. Matrix-matrix multiplication

\[\begin{align} Y_{m \times l} &= A_{m \times n}X_{n \times l} \end{align}\] \[\begin{align} f(Y) : \mathbb{R}^{m \times l} \mapsto \mathbb{R} \end{align}\]

Using subscript-summation nnotation to derive the gradients \(\frac{\partial f}{\partial X}\) and \(\frac{\partial f}{\partial A}\)

\[\begin{align} \frac{\partial f}{\partial X_{qr}} = \frac{\partial f}{\partial Y_{ij}} \frac{\partial (AX)_{ij}}{\partial X_{qr}} = \frac{\partial f}{\partial Y_{ij}} \delta_{jr} A_{iq} = \frac{\partial f}{\partial Y_{ir}} A_{iq} = A_{iq} \frac{\partial f}{\partial Y_{ir}} \end{align}\] \[\begin{align} \frac{\partial f}{\partial X} = A^T \frac{\partial f}{\partial Y} \end{align}\] \[\begin{align} \frac{\partial f}{\partial A_{qr}} = \frac{\partial f}{\partial Y_{ij}} \frac{\partial (AX)_{ij}}{\partial A_{qr}} = \frac{\partial f}{\partial Y_{ij}} \delta_{iq} X_{rj} = \frac{\partial f}{\partial Y_{qj}} X_{rj} \end{align}\] \[\begin{align} \frac{\partial f}{\partial A} = \frac{\partial f}{\partial Y} X^T \end{align}\]

4.1.4. Elementwise matrix function

\[\begin{align} Y_{n \times m} = \omega(X_{n \times m}) \\ f(Y) : \mathbb{R}^{n \times m} \mapsto \mathbb{R} \end{align}\] \[\begin{align} \frac{\partial Y_{ij}}{\partial X_{qr}} = \begin{cases} \omega'(X_{qr}), & (i = q) \& (j = r) \\ 0, & (i \neq q) \| (j \neq r) \end{cases} \end{align}\] \[\begin{align} \frac{\partial f}{\partial X_{ij}} = \frac{\partial f}{\partial Y_{ij}} \omega'(X_{ij}) \end{align}\]

4.2. Convolution

We start with 1D convolution. \(f\) and \(g\) are both 1D signals. \(f\) has length \(n\), \(g\) has length \(m\). The convolution of the two signals is defined as

\[\begin{align} (f*g)_i = \sum_j^m g(j) f(i-j+m/2) \end{align}\]

Effectively, we are sliding the kernel \(g\) across \(f\). At each kernel position, we multiply the overlapping values of the kernel and the image and sum the results.

Convolution is associative, e.g. if we want to convolve \(f\) with kernels \(g\) and \(h\), then

\[\begin{align} f*g*h = f*(g*h) \end{align}\]

When the kernel sizes are much smaller than the input image size, convolving the kernels first reduces the amount of computation.
